# shared 레포지토리
공유된 라이브러리 / 파일 등을 넣습니다. 저작권이 없이 마음대로 사용가능합니다.
다만, 라이브러리는 라이브러리 형태로만 써주세요. 뭐.. 상관은 없습니다!


## 깃허브 제대로 사용법
### 1. Issues에 글쓰기
    제목과 내용으로 글을 쓰세요!
    글을쓰면 이슈로 등록이되고, 빠르게 확인할 수 있습니다.
    해결할 버그나 문제가 있다면 여기로!
### 2. Pull requests에 요청하기(추천)
    브랜치와 브랜치를 비교하는 기능이에요.
    B의 변경사항을 A에게 적용할 수 있어요.
    **사용법**
    1. main 브랜치의 원본 코드를 복사한다.
    2. 고칠 점이나 추가할 점을 변경한다.
    3. Pull requests에 가서 New requests를 누른다음, 왼쪽값을 main, 오른쪽을 임시 작업 브랜치로 해주세요
    4. Create new requests 를 눌러서 글을 대충 쓴다음, 업로드하면 이것도 이슈처럼 됩니다.
    5. 나중에 제가 승인하면 되는거죠.
